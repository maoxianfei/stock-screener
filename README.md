# 火地晋爆点 — A股周线卦象量化选股 Pipeline

基于易经六十四卦的 K 线形态选股系统。在周线维度检测「火地晋 → 水雷屯」卦象序列，再通过量价双规则日线确认，输出共振信号。

## 筛选流程

```
全市场 A 股 (~5200只)
    │
    ▼
┌─────────────────────────┐
│  一轮：周线卦象扫描      │  21根周K线 → 滑动窗口 → 检测 [火地晋(40)→水雷屯(17)]
│  命中率: ~21%           │
└───────────┬─────────────┘
            │ ~1100只
            ▼
┌─────────────────────────┐
│  二轮：量价双规则 AND    │  拉日K线 → 量+价两个条件同时满足
│  通过率: ~45%           │
└───────────┬─────────────┘
            │ ~500只
            ▼
        最终股票池
```

## 卦象形态

**火地晋(40) → 水雷屯(17)** = 7根周K线的阴阳序列：

```
火地晋: 阳 阴 阳 阴 阴 阴     (冲高回落 + 三连阴洗盘)
水雷屯: 阴 阳 阴 阴 阴 阳     (延续回调 + 底部反转阳线)
```

## 二轮筛选规则

| 规则 | 维度 | 条件 | 参数 |
|------|------|------|------|
| 规则1 | 量 | 最近25个交易日至少1天成交量 > 120日均量 | 放量确认 |
| 规则2 | 价 | 最新收盘价 ≥ 70日均线 × 0.95 | 允许-5%容差 |
| 通过 | — | **规则1 ∧ 规则2** 同时满足 | AND逻辑 |

## 快速开始

```bash
# 全流程：一轮扫描 + 二轮筛选
python pipeline.py

# 仅跑一轮周线扫描
python pipeline.py --step 1

# 仅跑二轮（自动取最新一轮CSV）
python pipeline.py --step 2

# 单股全流程诊断
python pipeline.py --single 600021

# 自定义并发数和价格容差
python pipeline.py --workers 12 --tolerance 0.95
```

## 输出文件

所有结果在 `output/` 目录：

| 文件 | 内容 |
|------|------|
| `一轮周线信号数据_火地晋水雷屯_YYYYMMDD.csv` | 所有一轮命中的股票 |
| `二轮双规则数据_火地晋水雷屯_YYYYMMDD.csv` | 通过量价双确认的股票 |

### CSV 字段说明

**一轮 CSV**：

| 列 | 含义 |
|----|------|
| 股票代码 | 6位代码 |
| 股票名称 | 中文简称 |
| 命中次数 | 回看窗口内信号触发次数 |
| 形态结束日期 | 每次命中的结束日期（多值用分号分隔） |
| 信号区间 | 卦象起止日期 |
| 间隔(根) | 卦象间K线跨度（0=紧挨） |
| 最新收盘价 | 周线最新收盘价 |

**二轮 CSV**：

| 列 | 含义 |
|----|------|
| 5周量比(%) | 最近25天最高日量 / 120日均量 |
| 线上下% | 收盘价偏离70MA的百分比 |
| 状态 | 通过 / 仅量不达标 / 仅价不达标 / 量价双杀 |

## 项目结构

```
火地晋爆点/
├── pipeline.py          # 统一入口，一轮+二轮全流程
├── hexagram_engine.py   # 卦象编码引擎，K线→六十四卦映射
├── data_fetcher.py      # A股K线数据获取（mootdx）
├── weekly_match_stats.py # 独立一轮周线扫描（向后兼容）
├── round2_filter.py     # 独立二轮量价筛选（向后兼容）
├── output/              # 输出目录
└── README.md            # 本文件
```
